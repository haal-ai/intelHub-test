# Team Rust Onboarding Experience Report

**Author:** engineering-leads  
**Date:** 2026-03-05

## Context
Five Java developers (3-8 years experience) were onboarded to Rust over a 3-month period starting December 2025.

## Key Findings

### Pain Points
- Borrow checker concepts took ~3 weeks to become comfortable
- Lifetime annotations remain confusing after 3 months
- Error handling patterns (Result/Option) feel verbose initially

### What Helped
- "Rust by Example" as primary learning resource
- Pair programming with experienced Rust developer
- Starting with CLI tools before tackling async code
- Weekly Rust study group sessions

### Productivity Timeline
- Week 1-2: Basic syntax, simple programs
- Week 3-4: Comfortable with ownership model
- Month 2: Productive on small features independently
- Month 3: Contributing to production codebase

## Recommendation
Budget 6-8 weeks for full onboarding. Start with synchronous CLI projects before introducing async/await patterns.
