# Rust Async Runtime Evaluation: Tokio vs async-std

**Author:** platform-team
**Date:** 2026-03-11

## Context
Evaluating which async runtime to standardize on for our Rust microservices.

## Comparison
| Criteria | Tokio | async-std |
|---|---|---|
| Ecosystem support | Excellent | Good |
| Performance | Very high | High |
| Learning curve | Moderate | Lower |
| Community size | Larger | Smaller |

## Recommendation
Standardize on Tokio — larger ecosystem, better library support, and more production references.
